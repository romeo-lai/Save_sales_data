# DBF Sales ETL Pipeline

A lightweight ETL (Extract, Transform, Load) pipeline for importing legacy POS sales data from DBF files into a MySQL database.

## Overview

This project was created to modernize historical sales data stored in DBF files generated by a legacy POS system.

The application automatically:

1. Scans a root directory for `CTI.DBF` files.
2. Reads DBF records using Big5 encoding.
3. Transforms the raw records into a standardized JSON format.
4. Uploads the data to a PHP API endpoint.
5. Stores the data in a MySQL database.
6. Archives successfully processed files into date-based folders.
7. Continues processing even if individual files fail.

---

## Features

* Recursive directory scanning
* Automatic discovery of `CTI.DBF` files
* Big5 character encoding support
* Configurable file locations and API endpoints
* Batch JSON upload to API
* MySQL data ingestion
* Automatic archiving of processed files
* Error handling and processing summary
* Suitable for large historical data migrations

## Configuration

Create a `config.ini` file:

```ini
[PATHS]
dbf_directory = C:\eod
archive_directory = C:\archive

[API]
base_url = http://localhost/PHP TEST/Data_input.php
```

---

## Database Schema

The API stores sales records in the `item_sales` table.

Example fields:

```sql
FCODE
DESC1
DESC2
QTY
UNITPRICE
DATE
TIME
```
